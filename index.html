MicroFighter

MicroFighter is a 2D two-player fighting game I built in Godot. Beyond the game itself, we also built a custom Arduino Mega R3 controller with push buttons and an analog stick.

Currently polishing gameplay and refining controller integration for a smoother player experience.

To connect hardware with gameplay, I wrote a Python bridge that reads Arduino serial input, interprets button masks and analog stick values, and converts them into player controls. The script supports two players, handles analog deadzones, and can send both Windows keyboard events and UDP messages for integration.

  • Built gameplay in Godot for a local two-player fighting experience
  • Helped create a custom Arduino Mega R3 controller with push buttons and an analog stick
  • Wrote a Python bridge that parses serial payloads, maps player actions, and translates hardware input into usable game controls

CardMap

CardMap is a React + Vite application that helps players find local Magic: The Gathering card inventory using ZIP or GPS-based search.

The platform connects frontend search workflows to Supabase-backed functions and store data, then presents results through inventory views, a store directory, and map-based context.

CardMap inventory search is backed by a data pipeline that keeps listings normalized and refreshed for local store discovery.

  • Implemented location-aware search flows with ZIP validation, GPS lookup, and radius filtering
  • Built reliability patterns like caching and retry/backoff to keep search interactions responsive
  • Integrated store directory and map experiences using Supabase RPCs and Postgres/PostGIS data

Project

CardMap Inventory Scraper

This scraper is the ingestion layer behind CardMap. It collects multi-store inventory, standardizes the records, and publishes searchable data for the frontend.

The pipeline is built for operational reliability with per-store partitioning, retry handling, deduplication before upsert, and run-level reporting for auditability.

  • Supports both BinderPOS/Shopify and TCGplayer Pro store scraping flows
  • Applies normalization and accuracy filtering before database writes
  • Finalizes each run by refreshing the search surface consumed by CardMap
